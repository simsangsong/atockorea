# Design System Rules

- Use centralized design tokens only.
- Separate Brand colors (Navy, Blue, Ocean) from Status colors (Success, Warning, Error, Neutral).
- Use semantic roles instead of raw hex values in components.
- Support mobile-first responsive layout.
- Button minimum height must be 44px.
- Use visible focus rings.
- Status must not rely on color alone.
- Use tabular numerics for prices, times, countdowns, seat counts.
- Motion must be subtle: 150ms-220ms ease-out, opacity/scale only.
- Prefer a premium-but-calm travel aesthetic over a techy or cyber aesthetic.
