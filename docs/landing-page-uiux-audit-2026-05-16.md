# 랜딩 페이지 UI/UX 감사 및 개선안

작성일: 2026-05-16  
대상: AtoC Korea 메인 랜딩 페이지 `/`  
관련 코드: `components/home/v2/*`, `components/home/v2/sections/*`

## 1. 결론

현재 랜딩 페이지는 사진, 신뢰 지표, 투어 추천 기능, 지역/스타일/인기 상품 섹션을 모두 갖춘 상태다. 시각적 품질도 기본 이상이다. 다만 첫 화면에서 사용자가 즉시 해야 할 핵심 행동이 충분히 앞에 나오지 않는다.

가장 큰 UX 문제는 다음 한 문장으로 요약된다.

> "투어를 골라주는 서비스"라고 말하지만, 실제로 투어를 골라주는 입력 도구는 첫 화면 아래에 있다.

따라서 개선의 핵심은 새로운 기능을 늘리는 것이 아니라, 이미 존재하는 기능의 우선순위와 화면 배치를 재정렬하는 것이다. 디자인 톤을 바꾸거나 앱 기능을 해치지 않는 선에서, 첫 화면 안에 "선호 입력 - 추천 시작 - 신뢰 근거"가 함께 보이도록 만드는 것이 가장 효과가 크다.

## 2. 현재 화면 구조

현재 홈 페이지 구조는 다음 흐름이다.

1. `HeroSection`
   - 대형 여행 사진
   - 시즌 칩
   - H1: "당신의 한국 하루, 직접 선별."
   - 서브카피: "스타일만 알려주세요 - 맞는 투어가 30초 안에 떠요."
   - 신뢰 지표: `4.9 avg rating`, `100K+ bookings`, `8 platforms`
   - 매칭 입력 카드

2. `DeferredBestMatchPreview`
   - 매칭 결과 미리보기 영역

3. `DestinationsShowcase`
   - 서울, 부산, 제주 지역 카드

4. `ChooseTravelStyle`
   - 스몰그룹, 프라이빗, 버스투어 스타일 카드

5. `FeaturedProductsShowcase`
   - 인기 투어 카드

6. `WhyAtockorea`
   - 신뢰/차별화 설명

7. `ProcessOperational`
   - 이용 흐름

8. `FinalCTA`
   - 마지막 전환 유도

9. `StickyHomeCta`
   - 하단 고정 CTA

구성 자체는 합리적이다. 문제는 "서비스의 가장 강한 차별점"이 첫 화면에서 주 행동으로 충분히 느껴지지 않는다는 점이다.

## 3. 핵심 진단

### P0. 첫 화면의 주 행동이 늦게 등장한다

현재 첫 화면은 사진과 감성 카피가 강하고, 매칭 입력 카드는 사용자가 조금 더 내려와야 명확히 만난다. 하지만 페이지의 약속은 "스타일만 알려주면 30초 안에 맞는 투어를 보여준다"이다.

사용자 입장에서는 다음 질문이 생긴다.

- 어디에 스타일을 입력하지?
- 추천 결과가 바로 나오는 건가?
- 그냥 투어 리스트 사이트인가, 컨시어지 서비스인가?

첫 화면에서 이 질문이 해소되지 않으면, 아래 섹션의 완성도와 관계없이 전환 효율이 떨어진다.

개선 방향:

- 히어로 사진은 유지하되, 매칭 입력 카드 또는 축약형 입력 CTA를 첫 화면 안으로 더 끌어올린다.
- H1, 서브카피, 입력 카드, 신뢰 지표가 하나의 전환 흐름으로 읽히게 만든다.
- 모바일에서는 사진 높이를 과하게 차지하지 않게 하고, 입력 카드 첫 줄이 화면 안에 보이게 한다.

### P0. 포지셔닝 메시지가 두 갈래로 읽힌다

현재 페이지는 동시에 두 가지 메시지를 말한다.

- "우리가 직접 선별해준다."
- "사용자가 스타일을 입력하면 시스템이 맞는 투어를 찾아준다."

둘 다 좋은 메시지지만, 첫 화면에서는 하나의 주 문장으로 정리되어야 한다. 지금은 에디토리얼 큐레이션과 AI/스마트 매칭의 무게가 약간 나뉘어 보인다.

개선 방향:

- 핵심 포지셔닝을 "한국 투어를 많이 스크롤하지 않고, 내 여행 스타일에 맞게 바로 좁혀주는 서비스"로 통일한다.
- "직접 선별"은 신뢰 근거로 남기고, "30초 추천"은 행동 유도로 둔다.

추천 카피 예시:

```text
내 여행 스타일에 맞는 한국 투어를 30초 안에.
```

보조 카피 예시:

```text
서울, 부산, 제주 투어를 끝없이 비교하지 않아도 됩니다.
원하는 분위기만 알려주면 맞는 일정부터 보여드릴게요.
```

### P0. CTA 계층이 선명하지 않다

현재 화면에는 다음 전환 요소가 동시에 존재한다.

- 히어로 내부 매칭 버튼
- 지역 카드
- 스타일 카드
- 인기 상품 카드
- 하단 고정 CTA
- 최종 CTA

이 구조는 기능적으로 풍부하지만, 사용자의 첫 행동을 분산시킬 수 있다.

개선 방향:

- 1차 CTA: "최적 매치 보기"
- 2차 CTA: "인기 투어 둘러보기" 또는 지역 카드 진입
- 하단 고정 CTA는 사용자가 히어로를 지나친 뒤부터 보이도록 조정한다.
- 첫 화면에서는 하나의 주 CTA만 가장 강하게 보이게 한다.

### P1. 신뢰 지표는 좋지만 맥락이 부족하다

`4.9`, `100K+`, `8 platforms`는 강한 숫자다. 다만 숫자만 보이면 사용자는 다음을 궁금해한다.

- 어떤 플랫폼 기준인가?
- 예약 수는 누적 예약인가?
- 평점은 어떤 투어의 평균인가?

개선 방향:

- 숫자 자체는 유지한다.
- 지표 라벨을 조금 더 자연어화한다.
- 지표 아래나 근처에 짧은 보조 문장을 추가한다.

예시:

```text
4.9 평균 평점
누적 100K+ 예약
8개 플랫폼 운영 경험
```

보조 문장:

```text
실제 예약 데이터와 운영 경험을 기준으로 투어를 추천합니다.
```

### P1. 매칭 결과의 "보상"이 늦다

사용자가 입력하면 무엇을 받게 되는지가 첫 화면에서 충분히 구체적이지 않다. "투어가 나옵니다"는 맞지만, 결과가 카드인지, 일정인지, 가격 비교인지, 추천 이유인지 상상하기 어렵다.

개선 방향:

- 첫 화면 또는 바로 다음 영역에 작은 결과 예시를 둔다.
- 실제 결과 API를 더 무겁게 만들 필요 없이, 정적인 예시 카드로도 충분하다.
- 예시에는 "추천 이유"가 들어가야 한다.

예시 구성:

```text
추천 예시
제주 동부 자연 코스
왜 맞나요: 걷기 적당함, 사진 포인트 많음, 하루 일정
```

### P1. 섹션 간 정보 흐름은 좋지만 편집 리듬이 길다

현재는 히어로 이후에 지역, 스타일, 인기 상품, 신뢰, 과정, 최종 CTA가 이어진다. 정보는 충분하지만 사용자가 "결정"으로 가기 전에 탐색 요소가 많다.

개선 방향:

1. 히어로에서 매칭 시작
2. 바로 결과 미리보기 또는 인기 추천
3. 지역 선택
4. 스타일 선택
5. 신뢰 근거
6. 최종 CTA

즉, "결과를 먼저 보여주고, 분류는 그다음"이 더 전환 친화적이다.

### P1. 모바일에서 첫 화면 정보 밀도가 아쉽다

모바일에서 사진은 감정적 인상을 만들지만, 랜딩의 목적은 빠른 선택이다. 사진이 너무 많은 세로 공간을 차지하면 사용자는 핵심 입력을 만나기 전에 스크롤해야 한다.

개선 방향:

- 모바일 히어로 사진 높이를 유지하되, 입력 카드 상단이 첫 화면 하단에 걸리게 한다.
- H1과 서브카피는 더 짧고 직접적으로 유지한다.
- 입력 카드의 기본 상태는 너무 닫혀 있지 않게 한다.
- 스타일 칩은 1줄 또는 2줄 안에서 안정적으로 보이게 한다.

### P2. 시각적 톤은 좋지만 "프리미엄 여행 도구"보다 "감성 여행 랜딩"에 조금 더 가깝다

사진과 카피의 감성은 강하지만, 사용자가 실제로 "내게 맞는 선택을 빠르게 해주는 도구"라는 인상을 받으려면 더 도구적인 요소가 위로 와야 한다.

개선 방향:

- 감성 이미지는 유지한다.
- 입력, 선택, 결과, 이유를 더 명확한 UI 흐름으로 보이게 한다.
- 장식적 요소를 추가하지 않고, 정보 구조만 다듬는다.

## 4. 권장 랜딩 구조

### 제안 구조 A: 전환 중심형

이 구조가 가장 추천된다. 현재 기능과 디자인을 최대한 유지하면서 전환 효율을 높일 수 있다.

```text
1. Hero
   - 사진 배경
   - 명확한 H1
   - 한 줄 설명
   - 매칭 입력 카드
   - 신뢰 지표

2. Match Result Preview
   - "이런 식으로 추천됩니다" 예시
   - 실제 매칭 결과가 있으면 결과 카드

3. Featured Tours
   - 이번 주 많이 선택한 투어

4. Destinations
   - 서울, 부산, 제주

5. Travel Style
   - 스몰그룹, 프라이빗, 버스투어

6. Why Atockorea
   - 운영 경험, 검증 방식, 예약 안정성

7. Process
   - 선택 - 예약 - 투어룸/안내 - 여행

8. Final CTA
```

장점:

- 사용자는 첫 화면에서 바로 행동할 수 있다.
- 매칭 서비스라는 차별점이 선명해진다.
- 기존 컴포넌트를 거의 유지하면서 순서와 밀도만 조정할 수 있다.

### 제안 구조 B: 탐색 중심형

브랜드 감성을 조금 더 유지하고 싶다면 이 구조도 가능하다.

```text
1. Hero
   - 사진
   - H1
   - 짧은 CTA

2. Compact Match Bar
   - 목적지, 스타일, 입력, 추천 버튼

3. Destinations
4. Featured Tours
5. Travel Style
6. Trust
7. Final CTA
```

단점:

- 현재와 비교해 개선 폭이 작다.
- "30초 매칭"이라는 약속이 여전히 약해질 수 있다.

추천은 구조 A다.

## 5. 첫 화면 개선 상세안

### 목표

첫 화면에서 사용자가 바로 이해해야 하는 것:

1. 이 서비스는 한국 투어 선택을 줄여준다.
2. 내 선호를 입력하면 추천을 받을 수 있다.
3. 예약/운영 경험이 있어 신뢰할 수 있다.

### 첫 화면 정보 우선순위

```text
H1
내 여행 스타일에 맞는 한국 투어를 30초 안에.

Subcopy
서울, 부산, 제주 투어를 끝없이 비교하지 않아도 됩니다.
원하는 분위기만 알려주면 맞는 일정부터 보여드릴게요.

Match Card
[제주] [서울] [부산]
"가볍게 걷고 사진 찍기 좋은 하루"
[소그룹] [프라이빗] [가족] [크루즈]
[최적 매치 보기]

Trust Row
4.9 평균 평점 | 100K+ 누적 예약 | 8개 플랫폼 운영 경험
```

### 피해야 할 방향

- 새로운 히어로 일러스트 추가
- 색상 체계 변경
- 과한 애니메이션 추가
- 카드 안에 또 다른 카드 중첩
- 모든 섹션을 동시에 강조
- 텍스트가 많은 설명형 랜딩으로 변경

## 6. 섹션별 개선안

### HeroSection

현재 강점:

- 실제 여행지 이미지가 첫 인상을 만든다.
- 시즌 칩은 시의성이 좋다.
- 신뢰 지표가 첫 흐름에 들어와 있다.
- 매칭 기능이 이미 구현되어 있다.

문제:

- 사진과 감성 문구가 주인공이고, 매칭 기능이 두 번째로 밀린다.
- 사용자가 입력 도구를 보기 전에 페이지를 탐색형 사이트로 오해할 수 있다.

개선:

- H1을 더 직접적인 문제 해결형 문장으로 변경한다.
- 매칭 카드를 첫 화면 안으로 올린다.
- 히어로 하단의 여백과 통계 행을 재조정한다.
- 매칭 카드 내 CTA를 유일한 primary 액션으로 둔다.

### DeferredBestMatchPreview

현재 강점:

- 매칭 결과가 홈 안에서 이어질 수 있는 구조다.

문제:

- 사용자가 입력 전에는 이 영역의 기대 효과가 충분히 보이지 않을 수 있다.

개선:

- 입력 전 상태에는 "추천 결과 예시"를 보여준다.
- 입력 후 상태에는 기존 실제 결과를 유지한다.
- 결과 카드에는 추천 이유를 반드시 표시한다.

### DestinationsShowcase

현재 강점:

- 서울, 부산, 제주라는 주요 선택지를 빠르게 보여준다.
- 이미지 기반 탐색이 쉽다.

문제:

- 매칭보다 지역 탐색이 먼저 강해지면, 서비스가 일반 투어 카탈로그처럼 보일 수 있다.

개선:

- 매칭 결과 미리보기 뒤에 배치한다.
- 각 지역 카드의 CTA를 primary가 아닌 secondary로 보이게 한다.
- 지역 카드는 "선택의 시작점"이지 최종 CTA가 아니게 한다.

### ChooseTravelStyle

현재 강점:

- 사용자가 스몰그룹, 프라이빗, 버스투어 차이를 이해할 수 있다.

문제:

- "페이스만 고르세요. 나머지는 우리가 합니다."는 좋지만, 히어로의 매칭 메시지와 겹친다.

개선:

- 히어로에서는 "스타일 입력"을 말하고, 이 섹션에서는 "스타일 비교" 역할로 분리한다.
- 각 카드에는 가격보다 "누구에게 맞는지"를 먼저 보여준다.

### FeaturedProductsShowcase

현재 강점:

- 실제 상품 카드가 있어 랜딩이 추상적으로 끝나지 않는다.
- 구매 가능한 콘텐츠로 바로 이어진다.

문제:

- 너무 늦게 나오면 사용자가 실제 상품감을 늦게 본다.

개선:

- 매칭 결과 프리뷰 바로 다음으로 올리는 방안을 검토한다.
- "이번 주 많이 선택한" 문구는 유지하되, 추천 흐름과 연결한다.

### StickyHomeCta

현재 강점:

- 페이지 하단에서도 전환을 회수할 수 있다.

문제:

- 첫 화면에서 이미 CTA가 많은 경우 시각적 경쟁이 생길 수 있다.

개선:

- 히어로를 지나친 뒤부터 노출한다.
- 문구는 매칭 CTA와 동일하거나 보완적으로 맞춘다.
- 모바일에서 본문 카드나 버튼을 가리지 않는지 확인한다.

## 7. 카피 개선 원칙

### 원칙

- H1은 브랜드 시가 아니라 사용자 결과를 말한다.
- 서브카피는 "무엇을 줄여주는지"를 말한다.
- CTA는 동사로 끝낸다.
- 신뢰 문구는 숫자와 운영 맥락을 함께 둔다.

### 추천 문구 세트

1안:

```text
H1: 내 여행 스타일에 맞는 한국 투어를 30초 안에.
Sub: 서울, 부산, 제주 투어를 끝없이 비교하지 않아도 됩니다. 원하는 분위기만 알려주세요.
CTA: 최적 매치 보기
```

2안:

```text
H1: 한국 투어, 많이 찾지 말고 맞게 고르세요.
Sub: 여행 스타일과 도시만 알려주면 지금 예약하기 좋은 투어부터 좁혀드립니다.
CTA: 내 투어 찾기
```

3안:

```text
H1: 오늘의 한국 여행을 가장 맞는 투어로.
Sub: 선호, 페이스, 목적지만 알려주세요. 검증된 투어 중 맞는 선택지를 보여드립니다.
CTA: 추천 받기
```

가장 추천하는 조합은 1안이다. 현재의 "30초" 약속과 가장 자연스럽게 연결된다.

## 8. 시각 디자인 개선 원칙

기존 디자인을 해치지 않기 위해 아래 범위를 지킨다.

- 기존 사진 자산 유지
- 기존 컬러 축 유지
- 기존 카드 스타일 유지
- 기존 라운드, 그림자, 타이포 계층을 크게 바꾸지 않음
- 새 섹션을 대량 추가하지 않음
- 핵심 컴포넌트의 순서와 밀도만 조정

개선 시 확인할 점:

- 첫 화면에서 H1, 입력 카드, CTA가 동시에 읽히는가
- 모바일에서 입력 카드가 너무 아래로 밀리지 않는가
- CTA가 2개 이상 같은 무게로 경쟁하지 않는가
- 사진이 텍스트 가독성을 해치지 않는가
- 섹션 제목 크기가 카드 내부 텍스트와 충돌하지 않는가

## 9. 성능을 고려한 UX 개선

사용자가 이전에 지적한 로딩 문제와 연결해, 랜딩 개선은 성능에 부담을 주지 않는 방식이어야 한다.

권장:

- 히어로에 새 대용량 이미지 추가 금지
- 첫 화면 위쪽에 클라이언트 상태를 과하게 추가하지 않기
- 매칭 결과 예시는 정적 UI 또는 이미 로드된 데이터 기반으로 처리
- 아래쪽 상품/목적지 데이터는 기존 지연 로딩 구조 유지
- 애니메이션은 `prefers-reduced-motion` 대응 유지

피해야 할 것:

- 첫 화면에서 여러 API를 추가 호출
- 새로운 슬라이더/캐러셀 라이브러리 도입
- 큰 배경 비디오 추가
- 스크롤 기반 애니메이션 과다 추가

## 10. 구현 단계

### Phase 1. 첫 화면 전환 구조 정리

목표:

- 매칭 입력이 첫 화면 안에서 더 빨리 보이게 한다.
- H1/서브카피를 전환 중심으로 다듬는다.
- CTA 계층을 정리한다.

작업 범위:

- `components/home/v2/sections/hero-section.tsx`
- 번역 키 또는 홈 카피 리소스
- 필요 시 `StickyHomeCta` 노출 조건 조정

검증:

- 데스크톱 1440px에서 첫 화면 안에 매칭 카드가 보이는지
- 모바일 390px 폭에서 CTA가 첫 스크롤 전에 보이는지
- 기존 매칭 동작이 그대로 작동하는지

### Phase 2. 매칭 결과 기대감 강화

목표:

- 입력 전에도 사용자가 받을 결과를 상상할 수 있게 한다.

작업 범위:

- `components/home/v2/DeferredBestMatchPreview.tsx`
- 매칭 결과 empty state 또는 preview state

검증:

- 입력 전 상태가 과장된 마케팅 카드처럼 보이지 않는지
- 입력 후 실제 결과 흐름이 깨지지 않는지

### Phase 3. 섹션 순서 및 CTA 밀도 조정

목표:

- 매칭 - 추천 - 탐색 - 신뢰의 흐름을 만든다.

작업 범위:

- `components/home/v2/HomeV2Page.tsx`
- `FeaturedProductsShowcase`
- `DestinationsShowcase`
- `ChooseTravelStyle`

검증:

- 기존 섹션이 사라지지 않았는지
- 상품 카드 링크와 지역 카드 링크가 유지되는지
- 페이지 전체 높이와 흐름이 어색하게 끊기지 않는지

### Phase 4. 모바일 QA

목표:

- 작은 화면에서 입력, CTA, 스크롤 흐름이 안정적인지 확인한다.

체크리스트:

- 390x844
- 430x932
- 768x1024
- 1440x900

확인 항목:

- 텍스트 겹침 없음
- 버튼 안 텍스트 줄바꿈 이상 없음
- 하단 고정 CTA가 주요 버튼을 가리지 않음
- 섹션 간 여백이 과도하지 않음
- 입력 후 매칭 결과 영역으로 자연스럽게 이동

## 11. 성공 기준

정성 기준:

- 첫 화면만 보고도 "내 스타일을 입력하면 투어를 추천받는 서비스"라는 점이 이해된다.
- 사용자가 첫 클릭으로 무엇을 해야 하는지 망설이지 않는다.
- 사진, 신뢰, 상품, 매칭 기능이 서로 따로 놀지 않는다.
- 기존 브랜드 감성과 프리미엄 톤이 유지된다.

정량 기준:

- 히어로 CTA 클릭률 상승
- 매칭 입력 시작률 상승
- 첫 화면 이탈률 감소
- 인기 상품 카드 클릭률 유지 또는 상승
- 모바일에서 스크롤 깊이 개선

## 12. 적용하지 말아야 할 변경

이번 개선에서는 아래를 하지 않는 것이 좋다.

- 전체 디자인 시스템 교체
- 컬러 팔레트 대규모 변경
- 기존 상품 카드 UI 변경
- 투어 검색/매칭 로직 변경
- 새로운 데이터베이스 구조 도입
- 새 외부 UI 라이브러리 도입
- 랜딩을 설명형 긴 페이지로 변환

이 개선의 목적은 리디자인이 아니라, 현재 자산을 더 잘 팔리게 배치하는 것이다.

## 13. 최종 권장안

가장 먼저 할 일은 `HeroSection`의 정보 우선순위를 바꾸는 것이다.

우선순위:

1. H1을 결과 중심 문장으로 변경
2. 매칭 입력 카드를 첫 화면 안으로 이동 또는 더 위로 압축
3. 신뢰 지표를 자연어 라벨로 정리
4. `DeferredBestMatchPreview`에 입력 전 결과 예시 추가
5. `FeaturedProductsShowcase`와 `DestinationsShowcase` 순서 검토
6. `StickyHomeCta`가 첫 화면의 CTA와 경쟁하지 않게 조정

이 순서로 진행하면 기능과 디자인을 해치지 않고도 랜딩의 핵심 전환 흐름을 가장 크게 개선할 수 있다.
