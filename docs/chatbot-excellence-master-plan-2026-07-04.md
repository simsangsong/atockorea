# 챗봇 Excellence 마스터플랜 — 극한의 고객 체험 · 극한의 기능

작성 2026-07-04. **이 문서가 챗봇 트랙의 단일 기준(SoT)** — `chatbot-agent-roadmap-master-plan-2026-06-23.md`(5트랙 로드맵)과 `chatbot-checkout-funnel-master-plan-2026-06-23.md`(Q0–Q5)를 승계·통합한다. 두 문서는 이력 참조용.

## §A 목표와 원칙

**극한의 고객 체험** = ① 죽지 않는다(가용성) ② 거짓말하지 않는다(정확성·최신성) ③ 기다리게 하지 않는다(속도) ④ 손가락 하나로 끝난다(조작성) ⑤ 대화가 예약으로 이어진다(전환).

**극한의 기능** = 상담(문의·정책·POI) → 추천(매처) → 견적(슬롯→가격) → 예약생성(PENDING) → 체크아웃 링크 → 예약조회(본인확인) → 사람 핸드오프까지 **풀퍼널이 한 대화창에서** 끊김 없이 작동.

원칙:
1. **정확성 > 기능 수.** 스테일 데이터로 답하는 기능은 없느니만 못하다.
2. 가격·가용성·예약은 **결정론 엔진**이 답하고, 모델은 말투만 담당한다 (기존 아키텍처 유지).
3. 모든 웨이브는 **골든 테스트 배터리(§E)** 통과 후 종료 선언.
4. 쓰기(취소/변경/환불)는 기존 결정대로 관리자 승인 게이트 유지.

## §B 현재 구현 상태 맵 (2026-07-04 코드 기준)

파이프라인(순차): 검증 → 레이트리밋(20/min·200/hr IP) → **결정론 인텐트 분류**(10종, LLM 아님) → 조기게이트(핸드오프/예약조회/법률/견적) → 컨텍스트 조립(병렬: 상품·카탈로그·사이트지식·세션메모리·RAG·매처) → Gemini 2.5 Flash(temp 0.35, max 1200tok) → finalize(카탈로그 폴백·오라우팅 체크·에스컬레이션·메모리 갱신).

전부 구현 확인됨(✅): RAG 하이브리드(OpenAI 1536d 벡터+trigram RRF, top-8, sim≥0.18), 세션메모리(320자 PII-프리, chat_memory), 스트리밍 SSE(`CHAT_STREAMING=1` 게이트, 프로덕션 켜져 있음), 견적 Q0–Q4(슬롯수집 LLM→가격 결정론→PENDING 예약→`/itinerary-builder/checkout?bookingId=` 링크→위젯 결제버튼), 예약조회(이메일+예약번호, 열거방지 2/min), 에스컬레이션+Telegram, 어드민 analytics(볼륨·에스컬레이션·퍼널·피드백·커버리지갭·RAG 헬스).

주요 env: `GEMINI_API_KEY`, `OPENAI_API_KEY`(RAG), `CHAT_STREAMING`, `CHAT_RAG`, `CHAT_SESSION_MEMORY`, `PRICING_AUTOQUOTE_ENABLED`, `UPSTASH_*`(듀러블 리밋), `TELEGRAM_*`.

### Wave 2 — 똑똑한 라우팅 (인텐트·견적 플로우) ← C-9가 최우선

- **W2.1 (C-3·C-12)** 가격 질문 라우팅: `quote_request` 트리거를 "프라이빗/전세/차터+견적 신호"로 좁히고, **카탈로그 가격 질문 경로 신설** — 정가 표(카탈로그 컨텍스트에 USD 정가 명시 주입)로 숫자를 반드시 답하게 + "프라이빗 맞춤 견적도 가능" 한 줄. 골든테스트: 가격 질문 응답에 숫자 포함 필수.
- **W2.2 (C-10)** Q3 응답에 **예약번호(A2C-…) 포함** + "이 번호와 이메일로 언제든 여기서 예약 조회 가능" 안내. checkoutReadyReply에 reference 파라미터 추가.
- **W2.3** 견적 슬롯수집 UX: 자유 텍스트 대신 **위젯 내 칩/컨트롤**(지역 3버튼, 날짜 피커, 인원 스텝퍼, 시간 슬라이더) — 서버는 기존 슬롯 스키마 재사용. 오타이핑 제로로 견적 도달.
