## 확정된 사실 (재조사 불필요)
- 프로덕션 다운 원인은 Gemini 지출캡이었고 **해결됨**(사용자가 ₩30,000 상향). 프로덕션 챗봇 정상 작동 중.
- 아키텍처: 인텐트 분류(결정론 10종) → 조기게이트(핸드오프/예약조회/법률/견적) → 컨텍스트 조립(병렬: 상품·카탈로그·사이트지식·세션메모리·RAG·매처) → Gemini 2.5 Flash(temp 0.35) → finalize. 스트리밍 SSE는 `CHAT_STREAMING=1`로 프로덕션 켜져 있음.
- 가격·가용성·예약은 결정론 엔진, 모델은 말투만. **이 원칙 유지.**
- 라이브 DB = `mcp__atockorea__*`. 프로덕션 = https://www.atockorea.com. 메인 라우트 = `app/api/tour-product/assistant/route.ts`, 위젯 = `components/product-tour-static/_shared/TourProductAiAssistantWidget.tsx`.

## 구현 순서 (이 순서 고정 — 체감/위험 기준으로 재정렬됨)

**배치 1 — 즉효·고위험 차단 (먼저)**
1. **W0.8** 프론트 안전화 (C-34·C-35): SSE `JSON.parse`를 try-catch로 감싸 깨진 청크 스킵(위젯 무한 loading 방지) + `checkout_url`/링크를 `safeUrl()`(상대경로 or https 화이트리스트)로 검증 후 렌더. ← W4.0의 전제조건.
2. **W4.0** 마크다운 라이트 렌더러 (C-11): 위젯 메시지에 bold·리스트·링크만 화이트리스트 파싱(라이브러리 금지, XSS 안전). 내부 `/tour-product/…`는 same-tab, 외부는 새 탭. 지금 `**별표**`·`[링크](url)`가 원문 노출·클릭불가 상태.
3. **W2.0** 견적 멀티턴 stickiness (C-9): 인텐트를 마지막 메시지 단독이 아니라 대화상태로 — 직전 assistant가 견적/이메일 프롬프트면 후속 "네"/이메일을 견적 플로우로 계속 라우팅. + 모델 프롬프트에서 "예약 못해요" 자기부정 금지.
4. **W2.6** 과거날짜 검증 (C-18): `sanitizeDraft`에서 `requestedDate < todayISO`면 missing 처리 + 재프롬프트.

**배치 2 — 관측성·보안**
5. **W0.2 + W0.5** 실패 로깅 + 비용 텔레메트리 (C-2·C-14): chat_messages에 실패도 기록(error_code 구분) + Gemini usageMetadata로 tokens/cost/category 채우기 + 어드민 실패율·비용 타일 + 지출캡 접근 경보.
6. **W0.4** Gemini 폴백 (재발방지): 생성 실패 시 1회 재시도 → 카탈로그 폴백 or 정직한 장애안내+핸드오프. 빈 502 금지.
7. **W0.6 + W0.7** 히스토리 무결성 + 라이브티켓 인가 (C-29·C-30): assistant 턴을 확정/약속 소스로 취급 금지·로그는 서버 생성 reply만 / live 라우트 ticketId 소유권 검증.

**배치 3 — 최신성·정확도**
8. **W1.4 → W1.1** RAG 재색인 즉시 1회 실행 후 자동화 (C-4): 색인 시 비활성 19개 투어 청크 제외. (지금 6/22 색인이라 비활성 DMZ 투어 추천·구가격 노출 중)
9. **W2.1** 가격질문 라우팅 (C-3·C-12): 소그룹 가격질문을 견적 심문이 아니라 카탈로그 정가로 숫자 답변.
10. **W1.5.1** 과잉 에스컬레이션 차단 (C-20): 정보성 정책 질문은 티켓 미생성, 실제 요청만 티켓화.

이후 나머지(W2.2~2.10, W1.5.2~4, W3.x, W4.1~8, W6.x)는 플랜 §D 순서대로. 각 배치 끝나면 사용자에게 한국어로 진행보고.
