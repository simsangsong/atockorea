### §K B5 — QR 자동 체크인 + 환영 화면 → `212e837c`

🔴 **정적 QR에서는 자동이 성립하지 않는다.** 차량 외부 인쇄 QR은 인도에서도 스캔되므로,
자동이면 **타지 않은 사람이 탑승으로 기록**되고 시작 게이트가 빈자리를 안은 채 열린다 —
absent/노쇼 설계가 막으려던 바로 그 실패다. nonce 있는 콘솔 QR에서만 자동.
되돌리기는 **자동으로 된 것만** 가능(손님이 직접 누른 체크인·가이드가 해준 체크인은 유지 —
시작 게이트의 근거가 손님 쪽에서 흔들리면 안 된다).
화면은 질문이 아니라 환영이고, 막다른 화면이 아니라 투어룸으로 이어진다.

### §L — LLM 호출·토큰 최소화 (오너 지시 반영) → `c9162012` · `38e9b915`

플랜에 §L 트랙을 신설했다. **제약이 목표보다 강하다**는 것을 문서 첫 줄에 박았다:
절감안이 손님을 기다리게 하면 채택하지 않는다. 채택 가능한 절감은 셋뿐 —
ⓐ 더 빨라지면서 싸진다(캐시 히트) ⓑ 손님 경로 밖에서 미리 한다(프리워밍)
ⓒ 품질에 영향 없는 낭비를 제거한다.

먼저 **이미 있는 것부터 확인했다** — 번역 메모리·N로케일 1콜 배치·스킵 휴리스틱·Tier0 사전·
다이닝 셀 캐시·일일 예산 게이트·호출부 7곳 출력 상한. **재발명하지 않았다.**

구현한 것:
- **L0 계측** — `ops_ai_usage`. 라우터 **단 한 곳**에 심어서 8번째 호출부가 계측을 빠뜨릴 방법이 없다.
  fire-and-forget이라 응답 지연 **0ms**. §F의 "투어 1건당 30회"가 이제 **쿼리로 판정된다**(그동안 검증 불가였다).
  토큰을 **추정하지 않는다** — 추정 비용 그래프는 틀린 숫자를 확신하게 만든다.
  `skipped`(예산 방어)와 `failed`(장애)를 구분한다.
- **L1 출력 상한 기본값** — 생략이 "무제한"이 아니게. 기존 7곳은 명시값 우선이라 **바이트 단위 동일**.
- **L6 일일 예산 표면** — 오늘 쓴 호출·캐시로 아낀 호출을 **매일 1줄로** 보여준다. 예산 초과 투어만
  요주의로 올라온다(평상시 지출을 경고로 띄우면 경고를 안 읽게 된다). §F의 30회가 **알림이 됐다.**
- **L2 Tier1 응답 캐시** — 같은 투어 12명이 같은 질문 → **1콜**. 키에 컨텍스트 버전(스팟·라이프사이클·
  자유시간·30분 시각버킷)이 들어가 **지난 스팟의 답이 다음 손님에게 가지 않는다**.
  의미 유사도를 쓰지 않는 이유: 임베딩으로 판정하면 **호출 1회 아끼려고 호출 1회를 쓴다**.
  질문 원문은 저장하지 않는다.

기존 §14.4 목록에 더해 **이번 세션에서 새로 생긴 것**:

3. **개인 링크 재발송 정책 확인** — B0-D1c가 폐기-후-재발급이라 **재발송하면 먼저 받은 손님의
   기존 링크가 죽는다.** 의도된 동작이고 UI에도 표기했지만, 운영 중 재발송 빈도를 보고 판단하실 것.
4. **자동 체크인 실기기 QA** — 가이드 콘솔 QR(nonce)에서 스캔 즉시 체크인 + [수정] 되돌리기.
   정적 QR은 기존 탭 동작 유지가 맞는지 눈으로 확인.
