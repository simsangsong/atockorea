# 商家 Dashboard 进入方式

## 访问路径

### 商家登录页面
**URL：** `http://localhost:3000/merchant/login`

这是商家进入后台的第一步，需要输入邮箱和密码进行登录。

### 商家 Dashboard
**URL：** `http://localhost:3000/merchant`

登录成功后会自动跳转到商家管理后台。

## 访问流程

### 1. 首次访问（新商家）

1. **获取登录凭证**
   - 总台管理员会为商家创建账户
   - 商家会收到登录邮箱和临时密码

2. **访问登录页面**
   - 打开浏览器，访问：`http://localhost:3000/merchant/login`
   - 或点击总台提供的登录链接

3. **首次登录**
   - 输入邮箱和临时密码
   - 系统检测到首次登录，会强制要求修改密码
   - 设置新密码（至少8个字符）
   - 密码修改成功后自动跳转到 Dashboard

### 2. 日常访问

1. **访问登录页面**
   - 直接访问：`http://localhost:3000/merchant/login`

2. **登录**
   - 输入邮箱和密码
   - 点击"登录"按钮
   - 登录成功后自动跳转到 Dashboard

## Dashboard 功能模块

登录后可以访问以下功能：

- **대시보드** (`/merchant`) - 业务概览和统计数据
- **매출내역** (`/merchant/revenue`) - 销售和结算记录
- **상품관리** (`/merchant/products`) - 产品管理
- **주문관리** (`/merchant/orders`) - 订单管理
- **고객관리** (`/merchant/customers`) - 客户管理
- **데이터분석** (`/merchant/analytics`) - 数据分析
- **설정** (`/merchant/settings`) - 商家设置

## 移动端访问

商家 Dashboard 已优化为移动端优先设计，可以在手机、平板和电脑上正常使用：

- **移动端**：顶部菜单按钮，侧边抽屉式导航
- **桌面端**：固定侧边栏导航

## 注意事项

1. **权限要求**
   - 必须是商家角色（`role = 'merchant'`）
   - 商家账户状态必须为 `active`

2. **会话管理**
   - 登录后会话信息存储在 localStorage
   - 刷新页面不会丢失登录状态
   - 退出登录需要清除会话信息

3. **安全提示**
   - 首次登录必须修改临时密码
   - 建议定期更换密码
   - 不要在公共设备上保存登录信息

## 常见问题

### Q: 忘记密码怎么办？
A: 点击登录页面的"忘记密码？"链接，或联系总台管理员重置密码。

### Q: 登录后看不到数据？
A: 检查商家账户状态是否为 `active`，或联系总台管理员。

### Q: 移动端菜单打不开？
A: 确保浏览器支持现代 JavaScript，尝试刷新页面。
