## 인증키 안내

## 일반 인증키만 보일 때

공공데이터포털에서 **「일반 인증키」 하나만** 보이는 경우가 많습니다.  
그 키를 `.env`의 `TOUR_API_KEY`에 그대로 넣어 사용하면 됩니다.

- 우리 파이프라인은 `requests.get(..., params={...})` 로 호출하므로, **디코딩(원문) 키**를 쓰는 방식에 맞습니다.
- 포털에 표시되는 **일반 인증키**가 보통 그 디코딩 키이거나, 인코딩/디코딩 구분 없이 하나만 줄 때도 있어서, **지금 보이는 그 키 하나만 있으면 됩니다.**

---

## 인코딩/디코딩 키를 구분해서 보는 경우

일부 서비스나 화면에서는 **인코딩(Encoding)** / **디코딩(Decoding)** 이 따로 나옵니다.

- **찾는 위치**: [공공데이터포털](https://www.data.go.kr) 로그인 → **마이페이지** → **오픈API** → **인증키 발급현황** (또는 **개발계정 상세보기**) → 해당 API(예: 영문/중문/일문 관광정보서비스) 클릭 → 상세 화면에서 인증키 확인  
  - 같은 페이지에 **일반 인증키(인코딩)**, **일반 인증키(디코딩)** 이 둘 다 나올 수 있습니다.
- **우리 코드에서 사용할 키**:  
  - `serviceKey`를 **params**로 넘기고 있으므로 → **디코딩(Decoding) 키**를 사용하는 것이 맞습니다.  
  - 인코딩 키를 params에 넣으면 라이브러리가 한 번 더 인코딩해서 오류가 날 수 있습니다.

지금은 **일반 인증키 하나만** 있다면, 그 키만 등록해 두면 됩니다. 인코딩/디코딩이 따로 안 보이면 신경 쓰지 않아도 됩니다.

---

## .env 설정

```env
TOUR_API_KEY=여기에_포털에서_복사한_일반_인증키_붙여넣기
```

- 공백, 따옴표 없이 키만 넣습니다.
- 중·영·일·중문(번체) 등 **언어별로 다른 키를 쓸 필요 없습니다.**  
  동일한 인증키로 각 서비스(ChsService2, EngService2, JpnService2, ChtService2)를 호출합니다.

## 참고

- 2015년 1월 이후 발급된 인증키는 별도 인코딩 없이 사용하는 경우가 많습니다.
- `SERVICE_KEY_IS_NOT_REGISTERED_ERROR` 등 키 오류가 나면:  
  포털에서 **해당 API의 활용신청이 승인된 계정의 인증키**인지, 복사 시 앞뒤 공백이 들어가지 않았는지 확인해 보세요.

## API 에러 메시지 참고

| 에러 | 설명 |
|------|------|
| **Unauthorized** | 인증키 없음/무효 → 포털 인증키 확인 |
| **Forbidden** | 해당 API 활용신청/승인 없음 → 활용신청 여부·승인 상태 확인 |
| **API rate limit exceeded** | 동시 요청 한도 초과 → 잠시 후 재호출 |
| **API token quota exceeded** | 일일 호출 허용량 초과(429) → 초기화(다음날) 후 재이용 |
