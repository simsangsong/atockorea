## §A. 진단 — 현행 UI의 근본 문제 (감사 3트랙, 2026-07-19)

### A-1. 가장 큰 구조적 결함: **디자인 언어가 4개로 파편화**

| 패밀리 | 토큰/그래머 | 라이트/다크 | 성숙도 |
|---|---|---|---|
| **투어 손님 (tr-\*)** | `app/tour-room-theme.css` `--tr-*` (`.tr-root` 스코프) | ✅ `.dark .tr-root` | 성숙 (중립 잉크 확정) |
| **가이드 콘솔** | `tr-*` (`.tr-plan-root`) — 손님과 동일 | ✅ | 성숙 |
| **기사 콕핏** | ❌ **토큰 0** — raw `neutral-950/900/800`, 브랜드 hex 인라인, 이모지 | 다크 전용 | 파편 |
| **관제센터 (tour-ops)** | ❌ raw `slate-*` + `bg-white/5` + `.ops-light !important` remap, 별도 `opsTheme.ts` 엔진 | ✅ (tr-\*와 무관한 별도 엔진) | 파편 |
| **어드민 대시보드** | `--admin-*` 토큰 + 컴포넌트 킷 (`components/admin/*`) | ✅ navy 다크 | 성숙 (~80% 적용) |
| **마케팅 (home-v2 등)** | `home-v2*.css` 자체 시스템 + 각 마스터플랜 | — | 성숙·거버넌스 |

**핵심 진단:** "전체를 하나의 토큰으로 통일"은 **하지 않는다**(서로 다른 사용자·맥락: 손님/기사/관제/사장/방문객). 대신 **투어 운영 패밀리 내부의 3엔진(가이드 tr-\* / 콕핏 raw / 관제 slate)을 `tr-*`로 수렴**하는 것이 최대 구조 이득. 어드민·마케팅은 이미 성숙한 자체 시스템을 **완성**하는 방향(신규 시스템 도입 금지).

### A-2. 표면별 핵심 발견 (파일:라인 앵커)

**손님 앱 / 채팅방 (tr-\* — 이미 프리미엄, 밀도·구조 여지):**
- 팔레트 확정값 = **중립 잉크**(액센트 `#252a2c`, 내 말풍선 동일 잉크, 캔버스 소프트그레이 `#f3f4f2`). ⚠"딥 파인 말풍선/민트 캔버스"는 **폐기된 옛 방향** — 재도입 금지 (`tour-room-theme.css:9-11,42-43`).
- `ChatFeed.tsx:446-456` — 모든 말풍선 메타열에 **항상 노출되는 `IconMore` 버튼**이 가로폭·시각노이즈 유발 → 호버/롱프레스 시에만 노출(모바일은 `onContextMenu` 이미 있음). **밀도+마감 최대 이득.**
- `SpotArrivalCard.tsx:94-97` — 커버 이미지에 스크림/오버레이 타이틀 부재 → 그라디언트 스크림 = **프리미엄 최대 이득**(손님이 스크린샷하는 카드).
- `HomeTab.tsx:348,452` — 타일 `min-h-[86px]`+`gap-2` → `76px`+`gap-1.5`로 한 줄 회수(아이콘 칩 `h-11 w-11`=44px 유지).
- `RoomShell.tsx:383-432` — 스케줄 탭이 raw `<ol>`, 카드 재질 없음 → 카드화 + 활성스톱 elevation.
- 플랫 칩들(`Composer` 퀵리플라이 `:438`, `ConciergePanel` 칩 `:156`, `ChatFeed` 리액션 `:638`) → 헤어라인/`.tr-chip` 그라디언트로 재질 통일.
- 모션 저조 → tab pill/리액션/시그널 칩에 `active:scale` 스프링(Composer 전송버튼 수준으로).

## §B. 설계 결정 (U2-D 시리즈)

- **U2-D1 (구조 대원칙):** 4패밀리 유지, 강제 단일화 금지. **투어 운영 3엔진(가이드/콕핏/관제)만 `tr-*`로 수렴**. 어드민·마케팅은 자체 시스템 **완성**.
- **U2-D2 (컴팩트의 정의):** 컴팩트 = **패딩·갭·카드크롬·중복줄 축소**이지 **탭 타깃 축소가 아님**. 44px 하한은 전 패밀리 **불가침**. 세로공간은 "줄 병합·그룹리스트·아코디언"으로 회수.
- **U2-D3 (프리미엄의 정의):** 위계(타이포 역할 규율·tabular-nums) + 재질(플랫카드/칩그라디언트/elevation/overlay 4-tier) + 마이크로모션(스프링·스크림·리빌) + 마감(헤어라인·아이콘 일관성). **신규 색/장식 남발 아님.**
- **U2-D4 (팔레트 불변):** tr-\* = 중립 잉크(파인/민트 재도입 금지, SOS 레드·CaptionBanner 시네마틱 다크는 의도적 유지). admin = blue-600 단일 크롬 액센트. 마케팅 = 각 플랜 색 유지.
- **U2-D5 (모션 라이브러리):** 공용 마이크로모션 세트 정의 — `active:scale-95`(탭 피드백), 스크림 페이드, 리빌(메타/액션), reduced-motion 전면 킬. 저모션 시스템에 절제된 생동 추가.
- **U2-D6 (마케팅 게이트):** 마케팅은 "enrichment-compact"(정제·강화)만, 축소 금지. 착수 전 전용 감사 + 사용자 사인오프.

## §D. 권장 착수 순서 & 근거

1. **Wave 0 → Wave 1** 먼저. 이유: 그래머·토큰 수렴이 없으면 이후 프리미엄 작업이 3번 재작업됨(콕핏·관제 raw 위에 재질을 얹으면 낭비). 토큰 수렴 = 최대 레버리지 구조 이득이자 나머지의 전제.
